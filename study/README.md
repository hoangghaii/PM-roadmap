# Chương Trình Học Project Manager

> **Dành cho:** Software Developer muốn chuyển sang vai trò Project Manager  
> **Thời gian ước tính:** 3–6 năm để đạt Senior PM  
> **Cập nhật:** 2026

---

## Tại Sao Developer Nên Học PM?

Nếu bạn đang là developer và cảm thấy muốn có tầm ảnh hưởng rộng hơn — không chỉ viết code mà còn định hướng sản phẩm, điều phối đội nhóm, giao tiếp với khách hàng — thì con đường PM là một lựa chọn chiến lược rất phù hợp.

Developer chuyển sang PM có lợi thế cạnh tranh rõ ràng so với người không có nền kỹ thuật:

| Kỹ năng sẵn có từ Developer                           | Áp dụng vào PM như thế nào                                                              |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------- |
| Hiểu SDLC (Software Development Lifecycle)            | Lập kế hoạch sprint, ước lượng timeline chính xác hơn, biết task nào phụ thuộc task nào |
| Đọc hiểu code & nhận biết technical debt              | Giao tiếp tốt hơn với dev team, không bị "mù" kỹ thuật khi đàm phán scope               |
| Đã quen Agile/Scrum từ phía developer                 | Chuyển vai trò sang Scrum Master / Agile PM tự nhiên hơn người ngoài kỹ thuật           |
| Hiểu cách hệ thống vận hành, kiến trúc phần mềm       | Risk assessment kỹ thuật sâu hơn — phát hiện rủi ro mà non-tech PM bỏ sót               |
| Kỹ năng logic, tư duy hệ thống, phân rã vấn đề        | Phân tích scope, break down công việc, xử lý dependencies phức tạp                      |
| Quen với môi trường bất định, thay đổi liên tục       | Xử lý scope creep và thay đổi requirement hiệu quả hơn                                  |
| Làm việc với bug tracker, task board, version control | Sử dụng PM tools (Jira, Confluence, Notion) không mất thời gian học công cụ             |

## Hướng Dẫn Sử Dụng Tài Liệu Này

### Nếu bạn là Developer mới bắt đầu tìm hiểu PM

Bắt đầu từ [00-foundation/](./00-foundation/README.md). Đọc theo thứ tự từ file 01 đến hết. Làm tất cả bài tập trong folder `exercises/` trước khi chuyển sang giai đoạn tiếp theo.

### Nếu bạn đã có kinh nghiệm team lead / tech lead

Bạn có thể skip nhanh qua 00-foundation và vào thẳng [01-junior-pm/](./01-junior-pm/README.md). Tuy nhiên hãy đọc file [Iron Triangle](./00-foundation/02-iron-triangle.md) vì nó là nền tảng tư duy PM quan trọng nhất.

### Nếu bạn đang chuẩn bị thi chứng chỉ

Xem [04-certifications/](./04-certifications/README.md) để biết PMP hay CAPM phù hợp hơn với giai đoạn hiện tại của bạn.

Bạn có thể skip nhanh qua 00-foundation và vào thẳng [01-junior-pm/](./01-junior-pm/README.md). Tuy nhiên hãy đọc file [Iron Triangle](./00-foundation/02-iron-triangle.md) vì nó là nền tảng tư duy PM quan trọng nhất.

Xem [04-certifications/](./04-certifications/README.md) để biết PMP hay CAPM phù hợp hơn với giai đoạn hiện tại của bạn.

### Cách học hiệu quả nhất

1. **Đọc lý thuyết** từ các file `.md` trong từng thư mục
2. **Làm bài tập** trong thư mục `exercises/` — không bỏ qua bước này
3. **Áp dụng ngay** vào công việc hiện tại: quan sát PM ở công ty bạn, so sánh với lý thuyết
4. **Ghi chép reflection** sau mỗi sprint hoặc dự án: PM đã làm gì đúng/sai?
5. **Networking** với PM thực tế qua LinkedIn, meetup, hoặc nội bộ công ty
